2. 生成されたカンプ画像を `_work/comp/` 配下に保存（ファイル名例: `comp_hero_dark.png`）
3. カンプを Claude Code に共有 → Step 4 分析へ

## プロンプト本文（ChatGPT にコピペ）

```text
AIエンジニアのポートフォリオサイトのデザイン確認用カンプを作成してください。

【目的】
この画像は最終 LP ではなく、HTML/CSS 実装前にデザイン方向性を確認するためのビジュアルカンプです。

【作りたいもの】
ポートフォリオサイトを構成する各セクションを、複数枚の横長画像として作成してください。
それらを縦に並べると、1 本のポートフォリオサイトデザイン案に見える構成にしてください。

重要：ダーク系とライト系の2パターン作成してください。同じセクション構成で配色だけ変える。

【対象セクション】
1. Hero（キャッチコピー + サブテキスト。背景は抽象的な幾何学パターンか薄いグラデーションメッシュ）
2. About（2〜3行のプロフィール文のみ。シンプル）
3. Projects メイン（Bento Grid 風の大小混在カード。以下8種のカードを混在させる）
4. Projects 補助（小さい横並びリスト or タグ形式）
5. Skills（技術スタックをグループ化した列挙。バーグラフ不要）
6. Timeline（縦線タイムラインで経歴を表示）

【サイトの目的と閲覧者】
AIコンサル・DX推進系企業の採用担当者・技術責任者向け。
「営業現場の課題感 × AIエージェント実装スキル」の掛け合わせを視覚的に伝えるポートフォリオ。
PC 閲覧想定（スマホは最低限）。CTA なし・閲覧専用。

【Projects セクションの詳細】
Bento Grid 風に大小混在カードを並べる。大カードは 2〜3 列幅、小カードは 1 列幅。
各カードには以下のビジュアルを組み込む（カードの中に小さなフロー図や数字を入れる）：

- GitHub漏洩認証情報 検知（大カード）: スキャン→検知→通知のフロー図
- 通話記録 自動要約（大カード）: 録音→文字起こし→話者分離→要約→保存のフロー図
- AI会話 RAG基盤（大カード）: FTS5+ベクトル検索のアーキテクチャ図、「3,498セッション」という数字を大きく
- Webマーケティング内製化（中カード）: 「LP 9本」「チラシ 1,000枚」という数字グラフ
- フリマ6サイト管理（中カード）: 多サイト管理のダッシュボード風UIモック
- 法人営業アウトリーチ（小カード）: シンプルなフロー図、WIPラベル付き
- 日記自動記録（小カード）: API統合フロー図
- 自律AIエージェント基盤（中カード）: 4プロジェクトを束ねる構成図

【トーン】
- 目指すトーン: 信頼感・技術的な誠実さ・現場感・洗練（誇張なし）
- 避けるトーン: 「AIが何でもできる」系の誇張・過剰なハイテク感・ぎらついた装飾
- 参考イメージ: Brittany Chiang（洗練・高コントラスト）/ Bento Grid（モジュラー・情報密度）の中間

【配色 — ダーク案】
- 背景: #0a0f1e（深ネイビー）
- メイン: #7fadee（スカイブルー）
- アクセント: #5fe0a5（エメラルドグリーン）
- テキスト: #e8f0fe（ほぼ白）

【配色 — ライト案】
- 背景: #f8f9fc（オフホワイト）
- メイン: #1a4fa0（ロイヤルブルー）
- アクセント: #00b86b（グリーン）
- テキスト: #1a1f36（ほぼ黒）

※ HEX は参考値。最適な配色に調整してよい。紫グラデーションは避ける。

【デザイン方針】
- 余白を広めに取る（セクション間 100〜120px 相当）
- 高コントラストのタイポグラフィ（見出しと本文の重さに差をつける）
- フロー図・アーキテクチャ図はフラット風 + ライン系（過剰な3D・シャドウ・グローは避ける）
- 数字（266件・3,498セッション等）は大きく目立たせる
- 控えめなホバー効果・スクロールリビールが似合う静的なレイアウト

【画像・イラスト方針】
- フロー図の矢印・アイコンはシンプルなフラットスタイル
- 写真は使わない（AIアーキテクチャは写真で表現できない）
- 密度は適度（1カードに1ビジュアル）

【文字について】
- 画像内の日本語テキストは最小限（見出しや数字のダミーテキスト程度）
- 重要な本文は最終的に HTML で実装する前提
- 実在する個人名・会社名は入れない

【避けたいもの】
- 過剰なパーティクルアニメーション効果
- 「ロボット」「脳」「回路基板」のようなクリシェなAIアイコン
- グラデーションの多用（特に紫系）
- 全カード同サイズの羅列
- 意味のないデータビジュアライゼーション風装飾
- 過剰なグロー・光彩エフェクト
- 人物写真・ロボットAIのイラスト

【出力】
1. ダーク版: Hero / Projects（Bento Grid） / Timeline の3セクション
2. ライト版: Hero / Projects（Bento Grid） の2セクション
3. 実績カード拡大例（フロー図入りカードのイメージ 1〜2件）

各セクションを横長の Web デザインカンプとして作成してください。
全体として統一感があり、縦につなげると 1 本のポートフォリオサイトに見えるようにしてください。
```
